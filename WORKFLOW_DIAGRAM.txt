BRANCHING STRATEGY (Advanced)
==============================

main (production)
  |
  |-- develop (development)
  |     |
  |     |-- feature-new-ui
  |     |-- bugfix-post-timing
  |
  |-- hotfix-critical-bug


Workflow with branches:

1. Create feature branch:
   git checkout -b feature-new-ui

2. Work on feature:
   git add .
   git commit -m "Add new UI"

3. Push feature:
   git push origin feature-new-ui

4. Merge to main when ready:
   git checkout main
   git merge feature-new-ui
   git push origin main

5. Users update from main:
   updater.exe (pulls from main branch)
